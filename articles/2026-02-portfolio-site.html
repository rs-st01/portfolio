AI活用事例をまとめるポートフォリオサイトの設計・コーディング・記事作成までCursorで一括対応

Cursor GitHub Pages
サイト制作 ポートフォリオ 社内ツール

この事例のポイント

いま見ているこのポートフォリオサイト自体が、Cursor で制作した事例 です。

サイトの設計・HTML/CSSのコーディング・ビルドスクリプトの作成・GitHub Pages へのプッシュまで、すべて Cursor 上で完結しています。さらに、各記事の作成もCursorが担当しています。

Cursor が担当していること

1. サイト全体の設計・コーディング

  • 静的サイトジェネレーターの構成設計（Markdown → HTML のビルドパイプライン）
  • HTML テンプレート・CSS のコーディング
  • Python のビルドスクリプト（build.py）の作成
  • レスポンシブ対応

2. GitHub へのプッシュ

  • git add → commit → push まで Cursor のターミナルから一括対応
  • GitHub Pages でそのまま公開される構成

3. 記事の作成

各事例の記事も Cursor が作成しています。私がやっているのは以下だけです。

  • 口語に近いテキストで内容を羅列する（「こういう案件で、こういうツール使って、こういう成果が出た」レベル）
  • スクリーンショットや動画ファイルを貼る

これだけ渡せば、Cursor が内容を整理し、読みやすい構成に整えて、Markdownの記事ファイルとして完成させてくれます。

スピード感

サイト構築

仕様を決めるまでは少し時間がかかりましたが、仕様が固まってからは：

  • サイト全体の設計・コーディング・GitHub へのプッシュまで：10〜20分以内

テンプレート構成、CSS、ビルドスクリプト、初回デプロイまで含めてこの時間です。

記事作成

  • 1記事あたり：長くても10分程度で完成

口語テキスト＋スクショを投げるだけで、見出し構成・説明文・画像の配置まで整えてくれるので、体感としては「しゃべったら記事ができてる」に近い感覚です。

なぜ Cursor を選んだのか

今回のようなポートフォリオサイトは 完成形のイメージが明確 なプロジェクトです。「こういうデザインで、こういう構成のサイトが欲しい」というゴールが最初から見えている。

GPT に相談したところ、完成形が見えているクリエイティブ制作には Claude Code より Cursor のほうが向いている とのアドバイスがあり、今回は Cursor を採用しました。

実際、Cursor はコードの生成だけでなく、ファイル操作・ターミナル操作・git 操作まで一貫してこなしてくれるので、IDE の中だけですべてが完結するのは大きな強みでした。

成果

  • サイト構築（設計〜デプロイ）が 20分以内 で完了
  • 記事作成が 1本10分以内 で完了（口語テキスト＋スクショを渡すだけ）
  • デザイン・コーディング・記事執筆・デプロイまで ワンストップで Cursor に任せられる ことを実証
  • 社内のAI活用事例を素早く共有できる基盤が整った

再利用可能ポイント

  • Markdown + 静的サイトジェネレーターの構成はシンプルなので、他のドキュメントサイトにも転用可能
  • 「口語テキスト＋スクショ → Cursor が記事化」というワークフローは、ブログ・報告書・社内ナレッジなど幅広く応用できる
  • GitHub Pages での公開は無料で、社内限定（noindex対応済み）のサイト運用に最適

CMS不要で更新できる

一般的なWebサイトでは記事を更新するためにCMS（WordPressなど）を導入することが多いですが、このサイトではCMSは使っていません。

Cursor の Workspace でチャットするだけ で記事が次々と追加・更新されていきます。口語テキストとスクショを渡せばCursorが記事を整え、ビルドし、GitHubにプッシュするところまで一気通貫。CMS導入・管理のコストがゼロで、サイト更新が圧倒的に楽です。

課題・改善点

  • 仕様が曖昧な段階だと、Cursor とのやりとりに時間がかかる（完成イメージを先に固めるのが重要）
  • デザインの細かなニュアンス（余白感・フォントの雰囲気など）は、言語化が難しく微調整が発生しがち
